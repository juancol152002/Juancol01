# Explicación del Proyecto "Juancol01"

Este archivo detalla la estructura y funcionamiento del proyecto, dividido en Backend (Django) y Frontend (React/Vite).

## 1. Backend (Django)
El backend se encuentra en la carpeta `backend-proyectolenguaje3`. Es una API REST construida con Django y Django REST Framework.

### Estructura Principal
- **`paginacripto/`**: Configuración principal del proyecto (`settings.py`, `urls.py`).
- **`catalogo/`**: Aplicación para gestionar las criptomonedas y sus historiales de precios.
  - `models.py`: Modelos `Cripto` y `HistorialPrecio`.
  - `views.py`: ViewSets para exponer estos datos vía API.
- **`users/`**: Gestión de usuarios y autenticación.
  - Usa el sistema de autenticación de Django junto con Tokens (DRF).
- **`wallet/`**: Gestión de billeteras y transacciones de los usuarios.
  - `models.py`: Modelos `Wallet` y `Transaccion`.
  - Permite a los usuarios recargar saldo, comprar/vender criptos y ver su historial.

### Base de Datos
- Se utiliza **SQLite** (`db.sqlite3`) por defecto.
- **Usuarios Registrados**: Actualmente la base de datos cuenta con 2 usuarios principales (`luis.pv` y `Test`).

### API Endpoints Clave
- `/api/criptos/`: Lista de criptomonedas.
- `/api/users/registro/`: Registro de nuevos usuarios.
- `/api/token/`: Obtención de token JWT para login.
- `/api/wallet/dashboard/`: Información de la billetera del usuario autenticado.

## 2. Frontend (React + Vite)
El frontend está en `frontend-proyectolenguaje3/gestor-crypto`. Es una SPA (Single Page Application) moderna.

### Tecnologías
- **React**: Librería principal de UI.
- **Vite**: Empaquetador y servidor de desarrollo rápido.
- **TailwindCSS**: Framework de estilos para diseño moderno y responsivo.
- **Axios**: Para realizar peticiones HTTP al backend.
- **React Router**: Para la navegación entre páginas.

### Estructura Principal (`src/`)
- **`pages/`**: Vistas principales de la aplicación.
  - `LandingPage.jsx`: Página de inicio pública.
  - `AuthPage.jsx`: Login y Registro.
  - `UserDashboard.jsx`: Panel principal del usuario (ver saldo, gráficas, operaciones).
  - `AdminDashboard.jsx`: Panel de administración.
- **`components/`**: Componentes reutilizables (Navbar, Cards, Modales).
- **`App.jsx`**: Configuración de rutas y layout principal.

## 3. Notas Importantes
- **Autenticación**: El frontend guarda el token de acceso en `localStorage` tras el login y lo envía en los headers de cada petición (`Authorization: Bearer <token>`).
- **Despliegue**: Para producción, el frontend debe construirse (`npm run build`) y el backend debe configurarse con un servidor de producción (Gunicorn/Nginx) y desactivar `DEBUG=True`.
